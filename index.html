Glitchify — compare

Each demo: original WAV, then lofi at three intensities, then loop and wreck at intensity 100 (MP3). Files live in test/.

github.com/frangedev/glitchify

What you are hearing

  • Lofi 70 / 100 / 140 — same mode, more degradation as the number rises. Example: ./gfi --mode lofi --intensity 70 test/demo1.wav
  • Lofi 100 (sample) — same as default ./gfi; file name still uses _sample from the first batch.
  • Loop — needle-loop and skip character: ./gfi --mode loop --intensity 100 …
  • Wreck — heavier damage and reverse fragments: ./gfi --mode wreck --intensity 100 …

Locally: serve the repo root (e.g. python3 -m http.server). On GitHub Pages, deploy from branch main at the repository root.
